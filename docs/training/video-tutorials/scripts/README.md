# LLM Cost Ops Video Tutorial Scripts

## Overview

This directory contains production-ready scripts for the LLM Cost Ops video tutorial series. These scripts are designed to create comprehensive, professional training content that enables users to master the platform from basic setup through advanced enterprise deployment.

## Video Series Philosophy

The tutorial series follows a progressive learning path, starting with foundational concepts and building toward advanced enterprise scenarios. Each video is:

- **Standalone**: Can be watched independently with clear prerequisites
- **Practical**: Focuses on real-world scenarios and hands-on examples
- **Production-Ready**: Includes complete scene breakdowns, narration, and visual direction
- **Engaging**: Balanced mix of explanation, demonstration, and interactive examples
- **Accessible**: Clear language, multiple learning styles supported

## Production Notes

### Technical Requirements

#### Equipment
- **Screen Recording**: OBS Studio or Camtasia
- **Screen Resolution**: 1920x1080 (1080p minimum)
- **Frame Rate**: 30fps or 60fps
- **Audio**: Professional microphone (Blue Yeti, Shure SM7B, or equivalent)
- **Audio Format**: 48kHz, 24-bit WAV or 320kbps MP3

#### Software
- **Code Editor**: VS Code with recommended extensions
- **Terminal**: Modern terminal with syntax highlighting
- **Browser**: Chrome or Firefox (latest version)
- **Graphics**: Figma or Adobe After Effects for overlays

### Recording Guidelines

#### Visual Standards
- Clean desktop background
- Hide personal information
- Use large, readable fonts (16-18pt minimum for code)
- High contrast color schemes
- Consistent window sizes
- Smooth cursor movements
- Highlight cursor when needed

#### Audio Standards
- Clear, conversational tone
- Moderate pace (not too fast)
- Pauses for complex concepts
- Professional background (quiet room)
- Noise reduction in post-production
- Consistent volume levels

#### Screen Recording Best Practices
- Record in 1080p minimum
- Use 16:9 aspect ratio
- Show only relevant applications
- Close unnecessary notifications
- Use picture-in-picture for instructor (optional)
- Record in segments (easier editing)

### Post-Production Checklist

#### Video Editing
- [ ] Add intro/outro sequences
- [ ] Insert chapter markers
- [ ] Add lower thirds for speaker info
- [ ] Include code callouts and annotations
- [ ] Insert transitions between sections
- [ ] Add background music (subtle, royalty-free)
- [ ] Color correction and grading
- [ ] Export in multiple resolutions (1080p, 720p, 480p)

#### Audio Editing
- [ ] Noise reduction
- [ ] Equalization
- [ ] Compression
- [ ] Volume normalization
- [ ] Remove long pauses
- [ ] Add subtle background music

#### Subtitles/Captions
- [ ] Generate auto-captions
- [ ] Manual review and correction
- [ ] Add technical terms to dictionary
- [ ] Include code snippets in captions
- [ ] Support multiple languages (plan for translation)

### Branding Guidelines

#### Visual Identity
- **Color Scheme**: Use LLM Cost Ops brand colors
- **Logo Placement**: Top-right or bottom-left corner
- **Typography**: Consistent fonts for all text overlays
- **Templates**: Use standardized intro/outro templates

#### Voice and Tone
- Professional but approachable
- Enthusiastic about features
- Empathetic to user challenges
- Clear and concise explanations
- Active voice preferred

### Versioning Strategy

#### Version Numbering
Format: `v[major].[minor].[patch]`
- **Major**: Platform redesign or major feature changes
- **Minor**: New features or significant updates
- **Patch**: Bug fixes, clarifications, or minor improvements

#### Update Triggers
- Major platform releases (new version required)
- Significant UI changes (update affected videos)
- New features (create supplementary content)
- User feedback (patch existing content)

### Accessibility Requirements

#### Visual Accessibility
- High contrast text and backgrounds
- Readable font sizes (no text smaller than 14pt)
- Color-blind friendly palettes
- Descriptive alt text for graphics
- Avoid flashing or strobing effects

#### Auditory Accessibility
- Complete closed captions
- Transcripts available
- Clear audio without background noise
- Visual demonstrations don't rely solely on audio

#### Cognitive Accessibility
- Clear structure with chapters
- Consistent pacing
- Complex concepts broken into smaller parts
- Summaries at end of sections
- Downloadable resources and code samples
